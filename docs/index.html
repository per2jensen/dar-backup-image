// what you get

Everything you need. Nothing you don't.

A minimal, production-grade Docker image wrapping DAR and dar-backup for automated, verifiable long-term backups.

🗄️
Full · Diff · Incr
All three backup types supported. Run a complete FULL → DIFF → INCR chain, with one of each type permitted per definition per day.
🔐
Keyless Cosign Signing
Every release signed via Sigstore/Rekor using GitHub OIDC — no private key stored anywhere. Fully auditable transparency log.
🧬
SBOM Attestation
CycloneDX Software Bill of Materials generated by Syft and attached as a signed in-toto attestation to every release image.
🛡️
Grype Vulnerability Gate
Release is hard-blocked on any High or Critical CVE. Applies to both versioned releases and weekly :latest refreshes.
♻️
Weekly Image Refresh
:latest is rebuilt every Saturday, picking up Ubuntu base image patches. Tested, scanned, signed, and published automatically.
🔧
dar Compiled from Source
Since v0.5.15, dar is compiled from source with full zstd, lz4, Argon2, GPGME, and remote repository support. GPG-verified tarball.
🧩
par2 Included
Redundancy files generated automatically for bit-rot protection. Verify and repair archives years later.
👤
Non-root by Design
Pass RUN_AS_UID to run as your own user inside the container. Root (UID 0) is blocked by the helper script.
⏱️
CI / Cron / systemd Ready
Stateless, mount-based design. Drop into cron, systemd timers, GitHub Actions, or any container orchestrator without setup.

Quickstart

Pull the image and run your first backup in under a minute.

# Pull the latest signed image docker pull per2jensen/dar-backup:latest # Configure your directories export DATA_DIR=/home/alice/documents export BACKUP_DIR=/mnt/backups export RESTORE_DIR=/tmp/restore-test export BACKUP_D_DIR=/home/alice/.config/dar-backup/backup.d # Run a full backup docker run --rm \ -e RUN_AS_UID=$(id -u) \ -v "$DATA_DIR":/data \ -v "$BACKUP_DIR":/backups \ -v "$RESTORE_DIR":/restore \ -v "$BACKUP_D_DIR":/backup.d \ per2jensen/dar-backup:latest \ -F --log-stdout
# Or use run-backup.sh — handles all volume wiring for you WORKDIR=$HOME/dar-backup ./run-backup.sh -t FULL WORKDIR=$HOME/dar-backup ./run-backup.sh -t DIFF -d projects WORKDIR=$HOME/dar-backup ./run-backup.sh -t INCR
// mount configuration

Volume Mounts

How you map the source data directory into /data determines which backup definitions work — paths inside your definitions resolve relative to /data. The other mounts (/backups, /backup.d, /restore) are independent.

Host directory (default)Container mountPurpose
$WORKDIR/data/dataSource data to back up
$WORKDIR/backups/backupsDAR archives and log files
$WORKDIR/backup.d/backup.dBackup definition files
$WORKDIR/restore/restoreRestore verification target
# Scenario A — full host tree, back up multiple paths in one run docker run --rm -v /:/data -v "$BACKUP_DIR":/backups ... # definitions can target: data/home/alice/photos, data/srv/media, … # Scenario B — least-privilege, single directory only docker run --rm -v /home/alice/photos:/data -v "$BACKUP_DIR":/backups ... # definition: -R / -g data

Release Pipeline

Every image goes through a fully automated 10-step gauntlet before landing on Docker Hub. No manual push. No exceptions.

01
Build
dar-backup:dev built from source with versioned DAR_VERSION and DAR_BACKUP_VERSION pins. dar compiled from GPG-verified tarball.
02
Test Suite
Full pytest suite runs against the dev image. FULL → DIFF → INCR chain tested with SHA256 validation. Pipeline halts on any failure.
03
Promote
Tested dev image re-labeled to release tag. No rebuild — promoted image is byte-for-byte identical to the tested one.
04
Verify
OCI labels and embedded dar-backup --version checked against expected values. Mismatch aborts the release.
05
SBOM Generation
Syft generates a CycloneDX JSON Software Bill of Materials from the local image before it leaves the runner.
06
Grype Vulnerability Scan
Grype scans the SBOM. Any High or Critical vulnerability immediately aborts the release. Results uploaded to GitHub Security tab as SARIF.
Hard gate
07
Push to Docker Hub
Only reached after all gates pass. Publishes both :VERSION and :latest tags.
08
Cosign Keyless Signing
Image signed by digest using Fulcio certificate bound to GitHub OIDC. No private key stored anywhere.
Sigstore
09
SBOM Attestation
CycloneDX SBOM attached as a signed in-toto attestation via cosign. Verifiable by anyone without an account.
Rekor
10
Rollback Guard
If signing or attestation fails after the push, the image tag is automatically removed from Docker Hub.
# Verify any image yourself — no account required cosign verify per2jensen/dar-backup:latest \ --certificate-identity-regexp="https://github.com/per2jensen/dar-backup-image" \ --certificate-oidc-issuer="https://token.actions.githubusercontent.com"
